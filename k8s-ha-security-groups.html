03

Design decisions & gotchas

SG reference > CIDR

استخدم الـ Security Group ID كـ source مش 0.0.0.0/0. يعني الـ 6443 على الـ masters مصدره HAProxy-SG و Workers-SG نفسهم — كده لو الـ IPs اتغيّرت الـ rules تفضل شغّالة، والـ blast radius يفضل مقفول.

HAProxy = mode tcp

الـ 6443 لازم mode tcp (Layer 4 pass-through). لو عملته mode http هيكسر الـ mTLS بتاع الـ apiserver. الـ HAProxy بيعدّي الـ TLS زي ما هو من غير ما يفكّه.

Flannel port ≠ Calico

الرقم 8472/UDP ده خاص بـ Flannel VXLAN. لو غيّرت لـ Calico الأرقام تختلف: 179/TCP BGP و 4789/UDP VXLAN. اعرف CNI بتاعك قبل ما تقفل الـ SG.

Single LB = SPOF

الـ HAProxy الواحدة دي هي نفسها single point of failure. الحل production: keepalived + VIP بين HAProxyتين، أو تستخدم NLB مُدار على AWS بدل ما تعملها بإيدك.
